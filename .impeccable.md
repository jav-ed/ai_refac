# .impeccable.md

## Design Context

### Users
Developers and technical builders — comfortable with CLI tools, multiple languages, AI-driven workflows. They arrive at the docs site to evaluate or use `refac`. A secondary goal: the docs site is a trust signal that routes visitors to javedab.com for professional services. The audience must feel they are in expert hands.

### Brand Personality
**Trustworthy · Precise · Ethical**

The voice is knowledgeable but never arrogant. Respectful, thorough, and honest about limitations. Think: a senior engineer you'd want to hire — not flashy, not cold. Competent and kind.

### Aesthetic Direction
- **Minimal dark-first.** References: shadcn/ui, better-auth. Clean hierarchy, lots of breathing room, nothing decorative that doesn't earn its place.
- **Accent color: green** — not neon/lime (that screams AI gimmick), but a refined, desaturated green. Think forest, sage, or muted emerald. Professional and calm.
- **Dark mode is primary.** Light mode supported but dark is the intended experience.
- **Anti-references:** Glassmorphism, gradient text, glowing borders, cyan-on-dark, purple-blue gradients — all banned. Nothing that looks "AI-generated cool."

### Design Principles
1. **Clarity over decoration.** Every visual element must serve comprehension. If it doesn't help the reader, remove it.
2. **Earn trust through precision.** Tight spacing, consistent hierarchy, nothing sloppy. The design itself should communicate competence.
3. **Dark and calm.** Low visual noise. The code and content are the hero — the chrome steps back.
4. **Green as a quiet signal.** The accent is confident but not loud. It marks actions and focus states, not decoration.
5. **Cohesion with javedab.com.** The docs site and the main site should feel like they belong to the same person — related but not identical.
